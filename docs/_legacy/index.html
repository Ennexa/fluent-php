PHP extension for Project Fluent

Fluent translations with Rust-backed parsing for PHP.

Parse FTL resources, format localized messages, register PHP functions, and reuse parsed resources across requests in long-running PHP processes.

View Examples Build Extension
PHP API Bundles, resources, functions, exceptions
Rust core Uses Mozilla's Fluent implementation
Process cache Optional LRU cache for parsed resources
hello.php 
<?php

$resource = <<<'FTL'
hello = Hello, { $name }!
FTL;

$bundle = new FluentPHP\FluentBundle('en');
$bundle->addResource($resource);

echo $bundle->formatPattern(
    'hello',
    ['name' => 'John']
), PHP_EOL;

Use Cases

fluent-php keeps localization logic close to PHP while letting Fluent handle selectors, variables, and reusable message resources.

Applications

Localize product UI

Store copy in `.ftl` files and format messages with request data, user names, counters, and application state.

Content

Handle variants

Use Fluent selectors for statuses, flags, counts, and other branch-heavy messages without scattering conditionals in PHP.

Performance

Reuse parsed resources

Cache parsed FTL in each PHP-FPM worker so repeated requests avoid reparsing stable translation files.

Formatting

Call PHP functions

Register PHP callables for project-specific date, number, label, and domain formatting.

Validation

Surface parse errors

Catch parser and resolver exceptions with structured error details for tests, deploy checks, or admin tools.

Tooling

Preprocess FTL

Transform generated FTL strings before parsing; cache identity follows final source content.

Examples

These examples match the extension API: resources are parsed, attached to bundles, and then formatted by message id.

Variables Inject request data into localized messages. Selectors Choose message variants from values. PHP functions Expose application formatters to Fluent. Cached files Reuse parsed file resources inside a process.
variables.php 
$bundle = new FluentPHP\FluentBundle('en');
$bundle->addResource('welcome = Welcome, { $user }' . "\n");

echo $bundle->formatPattern('welcome', [
    'user' => 'Asha',
]);

Selectors keep copy rules in FTL

Instead of branching in controllers, keep the variants beside the translated text. Translators can adjust wording without PHP code changes.

selectors.php 
$bundle->addResource(<<<'FTL'
status = { $state ->
    [active] Account active
    [blocked] Account blocked
   *[other] Status unknown
}
FTL);

echo $bundle->formatPattern('status', [
    'state' => 'active',
]);

Use existing PHP formatters

Callables let messages use domain-specific formatting without forcing that logic into translation files.

functions.php 
$bundle->addResource(<<<'FTL'
today = Today is { FORMAT_DATE($date) }
FTL);

$bundle->addFunction('FORMAT_DATE', function (DateTimeInterface $date): string {
    return $date->format('Y-m-d');
});

Process-Local Cache

ResourceCache is designed for PHP-FPM and other long-running PHP processes. Each process owns its own independent LRU cache.

FTL file Load by canonical file path or source content.
ResourceCache Find a cached resource or parse a new one.
FluentResource Share the parsed resource across bundles.
FluentBundle Add resources for the request locale.
Message Format by id with PHP arguments.
cached-resource.php 
$resource = FluentPHP\ResourceCache::fromFile(
    __DIR__ . '/messages.ftl'
);

$bundle = new FluentPHP\FluentBundle('en');
$bundle->addResource($resource);

echo $bundle->formatPattern('checkout-title', []);

// After deploys or translation updates:
FluentPHP\ResourceCache::invalidateFile(
    __DIR__ . '/messages.ftl'
);
INI Setting Default Purpose
fluent.cache_enabled 1 Enable or disable ResourceCache.
fluent.cache_max_weight 16M Approximate total cache size.
fluent.cache_max_entry_size 2M Maximum approximate size for one entry.
fluent.cache_file_validation metadata Use metadata for speed or checksum for stronger change detection.

Invalidation after deleting a file is best-effort. The extension canonicalizes the nearest existing parent directory, which covers common symlinked prefixes such as macOS /tmp.

Build and Test

fluent-php currently builds as a Rust cdylib PHP extension. Load the produced library through PHP's extension ini directive.

terminal 
# Build debug extension
cargo build

# Run an example on Linux
php -d extension=target/debug/libfluent.so example/hello-world.php

# Run an example on macOS
php -d extension=target/debug/libfluent.dylib example/hello-world.php

# PHPT suite
php run-tests.php -n -d extension=target/debug/libfluent.so tests

Requirements

PHP 8.0 or newer, Rust 1.80 or newer, a C/C++ toolchain, and libclang headers for ext-php-rs.

Exception model

Parser, resolver, and cache errors extend FluentPHP\Exception, with structured details available from parser and resolver exceptions.
